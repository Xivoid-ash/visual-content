# visual-content
视觉考核 25电信3樊健良（edit阅读）
在这里我会逐一的讲述我做考核题的思路、想法、和最后的考核完成情况总结
1.赛事知识题和ROS2没什么好说的，学过就会
2.C++题我是直接用了两个类点类与装甲类为子父集，其他的就是输入数据与计算判断后按格式输出，没什么好讲的
3.opencv：1.第一题的图片很毛躁，调整了好一会的HSV和形态学开闭运算
          2.直接打开摄像头建立滑动条然后死循环就行，实现后发现帧率不对又锁定了一下帧率
          3.读取鼠标位置与RGB，等待点击然后记录点击位置与实时位置拉框，然后最后裁剪计算就行了
          4.苹果上色差大，调整好HSV后描绘轮廓用拟合外接圆框住就行
          5.正常的相机标定逻辑，没啥自己可以弄的思路
          6.图片就限制灯条的像素数，锁定到最小的那个装甲板，HSV预处理后滤波和形态学处理，灯条结构体去检测判断唯一装甲板，然后PNP解算距离，ROI框住装甲板然后锁定矩形中心后曝光提亮，为识别数字做准备（识别数字未做）
          视频逻辑也差不多只是从图片变成视频帧数，因为干扰少，所以适当减小滤波和形态学处理过程（识别数字未做）
          7.主要的时间都在调试能量机关了，虽然最后做的还是依托（test那份，另一份只能做到识别单个扇叶中心），不过还是将思路讲一下，最后我是准备用状态机去处理它，也设置了四个状态，识别中心R，锁定外一扇叶，收缩内外圆，跟踪扇叶旋转，先用滤波形态学HSV处理，状态一，而后找靠近视觉中心的最小拟合正方形，锁定完后定死中心R，而后进入状态二用中心R为圆心的广泛内外圆环ROI去锁定一个扇叶（最小二乘拟合圆）锁定完进入状态三后收缩内外圆半径至拟合圆的圆边，进入状态四，划分5个ROI矩形区在圆环内分别设为某一状态下5个扇叶对于中心R的角度位置，而后一个ROI区锁定到也绑定其余4个一起转动，设定扇叶结构体有三种状态，用像素数去确定扇叶状态为击中，未亮与未击打。旋转时输出转动半径与实时角度位置，最后用vofa+画图（状态二的bug太多，调整了两个多星期没效果）
4.YOLO:  1.手写识别直接找的letnet5使用教学里也有教这个模型与数据集
         2.用yolov8做的识别奶龙，效果还不错，我用的半自动打标，先手动打标随机1/5的数据集，训练出一个粗糙的模型让其帮我预测打标，最后自己再修改标定框，效果不错，自动打标基本都需要付费，数据集增强我用的opencv写个了随机曝光实际效果却和原本没啥差别
         3.yolo剪枝适合在较大的数据集里实现，我参考了b站上一些关于这个剪枝过程的视频，先训练一个模型VOC.yaml，而后稀疏训练，然后剪枝40%到60%最后恢复精度微调lrf和lr0就可以实现剪枝，不过稀疏训练我是用笨方式硬练，导致准备剪枝过程都要好久
考核完成情况总结：总体上都做完了，但是竞赛题确实缺少经验和好的想法而开源也基本上上了深度学习，导致拖了很长时间
